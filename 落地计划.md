# Catcat —— 基于 Mihomo 的 macOS 代理 App

## 一、技术选型

**核心方案：SwiftUI + Mihomo 内核（外挂二进制）**

| 层 | 选型 | 理由 |
|---|---|---|
| UI | SwiftUI (macOS 13+) | 原生动画流畅、声明式、性能好 |
| 状态管理 | Observation / TCA(可选) | 现代 Swift 数据流 |
| 内核 | mihomo 二进制 | 不重造轮子，直接复用成熟生态 |
| 通信 | RESTful + WebSocket（Mihomo External Controller）| 官方 API，热更新、流量推送 |
| 系统代理 | `networksetup` / SystemConfiguration framework | 设置 PAC/HTTP/SOCKS |
| TUN 模式 | NetworkExtension (PacketTunnelProvider) | 苹果官方方案，免 root |
| 配置解析 | Yams (YAML) | Swift 主流 YAML 库 |
| 持久化 | SwiftData / UserDefaults | 订阅、配置历史 |
| 打包分发 | Sparkle 2 | 自动更新 |

**为什么不用 Electron/Tauri**：体积大、动画卡顿、内存高，与"性能好+流畅动画"目标矛盾。SwiftUI 在 macOS 14+ 的动画引擎（Phase Animator、Keyframe Animator）足以做出极致体验。

## 四、UI/交互设计原则

- **简约**：留白克制、单层信息密度、强调主操作（开关/当前节点）
- **流畅动画**：
  - 节点切换 → `matchedGeometryEffect` 共享元素过渡
  - 延迟测试 → 数字滚动 + 颜色相位动画
  - 流量图 → `Canvas` + `TimelineView` 60fps 渲染
  - 模式切换 → spring(response: 0.4, dampingFraction: 0.8)
- **菜单栏**：使用 `MenuBarExtra(.window)` 自定义弹出面板（非系统菜单），可承载复杂动效
- **主窗口**：侧边栏 NavigationSplitView，参考 Linear / Things 3 的视觉语言

## 五、关键技术难点与方案

| 难点 | 方案 |
|---|---|
| mihomo 进程托管 | `Process` 启动，捕获 stdout/stderr；崩溃自动重启；退出时清理 |
| TUN 权限 | NetworkExtension System Extension，需 Apple Developer 账号 + Network Extensions entitlement |
| 系统代理无密码切换 | 使用 SMJobBless / SMAppService 注册 helper tool（特权辅助），或直接走 TUN 绕过 |
| 流量统计高频更新 | mihomo `/connections` WebSocket，UI 端节流 (throttle 500ms) + 增量 diff |
| YAML 配置兼容性 | 用 Yams 解析后保留原始字符串，避免重写丢注释 |
| 应用沙盒 vs 直分发 | 初期走 Developer ID 公证（非 MAS），保留全部权限 |

## 六、里程碑

按阶段推进，不绑定具体周期：

1. **M0 原型** ✅：内嵌 mihomo + 跑通 RESTful 调用，能切节点
2. **M1 MVP UI** ✅：菜单栏 + 订阅 + 节点列表 + 模式切换 + 流量图
3. **M2 系统集成** ✅：系统代理（Helper Tool 免密）、自启动、日志、连接管理
4. **M3 TUN 模式** 🚧：NetworkExtension 接入（待开始，需付费开发者账号）

6. **M5 发布**：Developer ID 签名公证 + 官网/GitHub Release（含 EdDSA 公钥与 appcast）

**当前进度**：M0–M2 已完成，下一步进入 M3 TUN 模式 / M4 打磨阶段。

## 八、风险提示

- **App Store 不可上架**（涉及代理/系统配置），走自分发
- **mihomo 协议**为 GPL/MIT 混合，封装时注意许可声明
- **NetworkExtension 需付费开发者账号**（$99/年）
- **Apple Silicon / Intel 双架构**：mihomo 二进制需打 universal
